HTML <template> Tag

Definition and Usage

The <template> tag is used as a container to hold some HTML content hidden from the user when the page loads.

The content inside <template> can be rendered later with a JavaScript.

You can use the <template> tag if you have some HTML code you want to use over and over again, but not until you ask for it. To do this without the <template> tag, you have to create the HTML code with JavaScript to prevent the browser from rendering the code.

HTML <fieldset> Tag

Definition and Usage

The <fieldset> tag is used to group related elements in a form.

The <fieldset> tag draws a box around the related elements.

Fruit juice size

When reading this form, a screen reader will speak "Fruit juice size small" for the first widget, "Fruit juice size medium" for the second, and "Fruit juice size large" for the third.

Disabled fieldset

The disabled attribute is a boolean attribute. A disabled fieldset is unusable and un-clickable. You can disable the entire fieldset or a single input.

HTML <datalist> Tag

Definition and Usage

The <datalist> tag specifies a list of pre-defined options for an <input> element.

The <datalist> tag is used to provide an "autocomplete" feature for <input> elements. Users will see a drop-down list of pre-defined options as they input data. They can select one of those suggested values or type in their own value.

Example

  • The <datalist> element's id attribute must be equal to the <input> element's list attribute (this binds them together).

  • You can disable each option, using the attribute disabled.

Note: The datalist tag is not supported in Safari 12.0 (or earlier).

HTML <select> Tag

Definition and Usage

The <select> tag specifies a list of pre-defined options for an <input> element.

Unlike <datalist> tag, <select> doesn't allow users to enter their own value.

Example

You can disable each option, using disabled attribute. A disabled option is un-clickable.

HTML <dd> Tag

Definition and Usage

The <dd> tag is used to describe a term/name in a description list.

The <dd> tag is used in conjunction with <dl> (defines a description list) and <dt> (defines terms/names).

Inside a <dd> tag you can put paragraphs, line breaks, images, links, lists, etc.

Example
Coffee
Black hot drink
Milk
White cold drink

HTML <details> Tag

Definition and Usage

The <details> tag specifies additional details that the user can open and close on demand.

The <details> tag is often used to create an interactive widget that the user can open and close.

Examples

By default, the widget is closed. When open, it expands, and displays the content within.
Closed by default
Epcot Center

Epcot is a theme park at Walt Disney World Resort featuring exciting attractions, international pavilions, award-winning fireworks and seasonal special events.

To be open by default - add open attribute. (<details open>).
Open by default
Epcot Center

Epcot is a theme park at Walt Disney World Resort featuring exciting attractions, international pavilions, award-winning fireworks and seasonal special events.

HTML <dialog> Tag

Definition and Usage

The <dialog> tag defines a dialog box or subwindow on top of the content.

The <dialog> element makes it easy to create popup dialogs and modals on a web page.

Example

This is an open dialog window

By default, the dialog box is closed. To open it - add open attribute. (<dialog open>). To open as modal - use showModal() method.

HTML <dfn> Tag

Definition and Usage

The <dfn> tag stands for the "definition element", and it specifies a term that is going to be defined within the content.

The nearest parent of the <dfn> tag must also contain the definition/explanation for the term.

Example

HTML <em> Tag

Definition and Usage

The <em> tag is used to define emphasized text. The text is typically displayed in italic.

A screen reader will pronounce the words in <em> with an emphasis, using verbal stress.

Example

You have to hurry up!

HTML <del> Tag & HTML <ins> Tag

Definition and Usage

The <del> tag defines text that has been deleted from a document. Browsers will usually strike a line through deleted text.

The <ins> tag defines a text that has been inserted into a document. Browsers will usually underline inserted text.

Example

My favorite color is blue red.

  • The cite attribute specifies a URL to a document that explains the reason why the text was inserted/changed.
  • The datetime attribute specifies the date and time of when the text was inserted/changed.
  • Both attributes has no visual effect, can be used by screen readers only.

The <output> tag is a container element into which can be injected the results of a calculation or the outcome of a user action.
